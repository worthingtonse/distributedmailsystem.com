================================================================================

CRITICAL ISSUE #1: INVALID/MOCK SUBSCRIPTION PLAN IDs
------------------------------------------------------
Location: src/pages/Subscribe.jsx (lines 38-40)

The code uses MOCK plan IDs that don't exist in PayPal:

    plan_id: selectedPlan.price === '10'
      ? (import.meta.env.VITE_PAYPAL_PLAN_ID_BASIC || 'P-MOCK-10')   // INVALID
      : (import.meta.env.VITE_PAYPAL_PLAN_ID_PRO || 'P-MOCK-20'),    // INVALID

The fallback values 'P-MOCK-10' and 'P-MOCK-20' are placeholder strings,
NOT real PayPal subscription plan IDs.

Real PayPal plan IDs look like: P-5ML4271244454362WXNWU5NQ


CRITICAL ISSUE #2: NO ENVIRONMENT CONFIGURATION FILE
-----------------------------------------------------
No .env file exists in the project root directory.

Without a .env file, all PayPal configuration falls back to mock values:
  - Client ID falls back to: 'sb' (generic sandbox)
  - Plan IDs fall back to: 'P-MOCK-10' and 'P-MOCK-20' (non-existent)


CRITICAL ISSUE #3: SUBSCRIPTION PLANS NOT CREATED IN PAYPAL
-----------------------------------------------------------
Even if you add a .env file, you must first CREATE the subscription plans
in the PayPal Developer Dashboard. The plan IDs in your code must match
actual plans created in your PayPal sandbox business account.

================================================================================
                           SOLUTION STEPS
================================================================================

STEP 1: Create PayPal Sandbox Business Account
----------------------------------------------
1. Go to: https://developer.paypal.com/
2. Log in with your PayPal developer account
3. Navigate to: Dashboard > Sandbox > Accounts
4. Create a Business sandbox account if you don't have one

STEP 2: Create Subscription Products and Plans
----------------------------------------------
1. Go to: https://developer.paypal.com/dashboard/applications/sandbox
2. Navigate to: Products & Plans (or use REST API)
3. Create a Product:
   - Name: "QMail Subscription"
   - Type: SERVICE
4. Create Plans under that Product:
   - Plan 1: "Starter Top-up" - $10/month
   - Plan 2: "Power User" - $20/month
5. Copy the Plan IDs (format: P-XXXXXXXXXXXXXXXXXXXXX)

STEP 3: Get Sandbox Client ID
-----------------------------
1. Go to: https://developer.paypal.com/dashboard/applications/sandbox
2. Click on your sandbox app (or create Default Application)
3. Copy the "Client ID" from the sandbox credentials

STEP 4: Create .env File
------------------------
Create file: .env in project root with:

    VITE_PAYPAL_CLIENT_ID=your-sandbox-client-id-here
    VITE_PAYPAL_PLAN_ID_BASIC=P-xxxxx-your-10-dollar-plan-id
    VITE_PAYPAL_PLAN_ID_PRO=P-xxxxx-your-20-dollar-plan-id

STEP 5: Restart Development Server
----------------------------------
After creating .env file, restart Vite:
    npm run dev
